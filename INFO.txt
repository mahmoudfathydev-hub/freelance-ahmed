### 2. AOS (ANIMATE ON SCROLL) ANIMATIONS

Default Configuration:
```javascript
AOS.init({ 
  once: true,           // Animate only once
  duration: 1000,       // 1 second animation duration
  easing: 'linear'      // Linear easing function
})
```

#### Animation Types & Usage:

**fade-down**
- Elements fade in while moving downward
- Applied to: Page headers, main titles, section headings
- Delay: 0-1000ms (sequential for multiple elements)
- Example: "My Projects", "Get In Touch"

**fade-up**
- Elements fade in while moving upward
- Applied to: Cards, content blocks, feature sections
- Delay: 100-400ms (incremental per item)
- Example: Project cards, contact info cards, FAQ items

**fade-left**
- Elements fade in while moving from the left
- Applied to: Sidebar content, right-aligned sections
- Delay: Sequential
- Example: Contact form sidebar

**fade-right**
- Elements fade in while moving from the right
- Applied to: Back buttons, left-aligned sections
- Delay: Sequential
- Example: Back to Projects button

## üöÄ PERFORMANCE CONSIDERATIONS

### Optimization Techniques
1. **Image Optimization** - Next.js Image component with lazy loading
2. **Code Splitting** - Automatic per-route splitting
3. **CSS Optimization** - Tailwind PurgeCSS removes unused styles
4. **Smooth Scroll** - Native CSS implementation
5. **AOS Trigger** - Once-only animations to reduce reflows

### Animation Performance
- CSS transitions used (GPU-accelerated)
- Transform properties used (scale, translate)
- Opacity changes (efficient)
- Avoided: Border width, box-shadow (causes repaints)
